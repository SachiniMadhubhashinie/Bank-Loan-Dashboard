# Bank-Loan-Dashboard
Developed an interactive loan analytics dashboard to track key KPIs such as total loan applications, funded amount, amount received, interest rate, and DTI. The project includes summary, overview, and detailed dashboards to analyze loan performance, borrower behavior, and regional trends for data-driven decision-making.

<h2>Project overview </h2>
<p>This repository contains a Bank Loan Analysis project built using SQL. The project focuses on analyzing loan applications, funded amounts, repayments, interest rates, and borrower financial health. It helps understand loan portfolio performance through KPIs, trends, and status-based insights.<br>

The project is ideal for beginners and intermediate learners to practice aggregation, filtering, grouping, date functions, and business analytics using SQL.</p>

## Objectives

1. **Analyze Loan Performance**: Track loan applications, funded amounts, and repayments.
2. **KPI Monitoring**: Calculate MTD and MoM metrics for key loan indicators.
3. **Risk Evaluation**: Compare Good Loans vs Bad Loans.
4. **Trend Analysis**: Identify monthly, regional, and borrower-based trends.
5. **Business Insights**:Support data-driven lending decisions.

## Project Structure

### 1. Database Setup

- **Database Creation**: The project starts by creating a database named `Bank_Loan_DB`.
- **Table Creation**: A table named `bank_loan_data` is created to store the sales data. The table structure includes columns for id,address_state,application_type,emp_length,emp_title,grade,home_ownership,issue_date,last_credit_pull_date,last_payment_date,loan_status,next_payment_date,member_id,purpose,sub_grade,term,verification_status,annual_income,dti,installment,int_rate,loan_amount,total_acc and total_payment.
